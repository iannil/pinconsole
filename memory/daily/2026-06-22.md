### 经验沉淀

- **Astro 5 output 选项变化**：`hybrid` 已删除，`static`（默认）现在等价于原 hybrid。看官方 changelog 而不是凭记忆写配置。
- **Astro 5 API 路由必须显式 `prerender = false`**：在 `output: 'static'` 模式下，API 路由默认被 prerender 为静态文件，POST 会 405。**所有 `src/pages/api/*.ts` 第一行加 `export const prerender = false;`**。本地 build 看不出问题（prerender 报警告但不报错），只有真发 POST 才暴露。
- **表单字段验证要容忍 undefined**：客户端可能省略可选字段（curl 测试最易暴露），`!=` 而不是 `!==` 更宽容（undefined == null 为 true），D1 bind 用 `??` 兜底。
- **Cloudflare adapter SESSION KV warning**：build 时会 warn SESSION binding 缺失，但目前不使用 session（无登录态），wrangler.toml 暂可不配 KV。如果后续需要 session，加 `[[kv_namespaces]] binding = "SESSION"`。
- **OSS 仓库与 maintainer 营销代码区隔**：`landing/`（OSS 模板，用户部署）vs `marketing/`（maintainer 营销站）。两者物理分目录 + 文档明示职责边界，避免未来 Claude/贡献者混淆。
- **`wrangler pages dev` 本地能完整跑 Cloudflare Workers runtime**：包括 D1 binding（`--local` 模式 SQLite 文件存 `.wrangler/state/v3/d1/`），是端到端验证 Workers + D1 流程的最快路径，不需要 Cloudflare 账号。
