### 关键设计取舍

- **1b 不集成 rrweb**：访客"看到"的是抽象事件（坐标、点击位置），不是真实页面回放。1b 验证管道，1c 验证可视化。
- **1b 不实现 co-browsing**：admin 只能"看"，不能"控制"。1e 起加双向通道。
- **1b 不做认证**：admin Web 端任何人可访问，仅 dev 环境安全。1h 起加 auth。
- **MessagePack vs JSON**：选 MessagePack 是因为 rrweb 事件流量大、运营端带宽敏感；debug 不直观是代价，但 envelope 字段固定、可读性可接受。
- **Discriminated union vs 全包结构**：前者类型安全 + 带宽友好 + LLM 友好，未来加事件类型只需扩 union，不破坏旧消费端。
- **Hub 内存索引 vs Redis presence**：选内存索引因为 hub 已经维护连接，索引是免费副产品；Redis presence 是冗余。
- **不引入 codegen（quicktype/buf）**：1b 协议简单，手写 Go + TS 即可；稳定后再 codegen。
