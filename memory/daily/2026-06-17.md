**与规格的偏差（5 项，全部已记录在完成报告）**：
1. `concurrently` 替代 `overmind`（避免 tmux 依赖）
2. Go 1.25 实际（spec 写 1.22+）
3. visitor-sdk 用 `tsc` 而非 `vue-tsc`（非 Vue 项目）
4. admin tsconfig 合并为单文件（composite + noEmit 冲突）
5. Gin 路由用 `NoRoute` 做 SPA fallback（catch-all 与 StaticFS 同前缀冲突）

**实施期间踩过的坑（值得记忆）**：
- Gin 的 catch-all wildcard（`/admin/*any`）与 StaticFS（`/admin/assets`）不能同前缀，必须用 NoRoute 做 SPA fallback
- /admin/assets/missing.js 缺失时会被 NoRoute 兜底为 SPA index.html，需要在 NoRoute 中显式判断 `/admin/assets/` 前缀返回 404
- Vue + Vite + TS 的 composite project references 与 noEmit 冲突，单 tsconfig 更简单
- pnpm test:js 默认递归所有 workspace，需要显式 filter 排除 e2e（否则无 server 时失败）
- Go module 在 tidy 时会自动调整 go directive 到当前工具链版本
