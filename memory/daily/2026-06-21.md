### 修复 1:P1-chat-render(visitor→chatWidget 收起时丢消息)

**根因**(systematic analysis 确认):
- `receiveMessage` 把 `lastMessageId` 当"已接收 + 已渲染"双重判据
- WS push 在 widget collapsed 时只 update unreadCount,但 `lastMessageId` 已被推进到 msg.id
- 展开 → `fetchMessages(lastMessageId)` → server 视角 `since_id=msg.id` 返回空 → 消息永久丢

**修法**:用两个独立 Set 分离"已收到"与"已渲染"
- `seenIds`(WS 层 dedup,避免重复 unread++)
- `renderedIds`(DOM 层 dedup,避免重复渲染)
- `fetchCursor`(fetchMessages 的 since_id,只随"已渲染"推进)

**TDD**:`visitor-sdk/tests/chat-screenshot.test.ts` 加 3 个新测试
1. "收起时收到 → 展开 → fetchMessages 拉回并渲染"(原 bug 场景)
2. "展开状态 WS + fetchMessages 不重复渲染"(race 场景)
3. "本地回声(负 id)+ server 消息(正 id)不冲突"

**GREEN**:visitor-sdk 214/214 测试全过(3 new + 211 existing)。

### 修复 2:P1-claim-TTL(5min TTL 无续期)

### 23:00 — replay 三 bug 修复(双重 .rr-player + iframe 150px 压扁 + initialized flag spam)

**起点**:用户报告 "录屏尺寸显示错误,大片的空白,只有一小块是用户页面录屏",提供 HTML 显示 `.player-container` 内有两个 `.rr-player` 元素。

**根因 1**:`ReplayPlayer.vue` 的 `rebuildPlayer` race condition。
- `onMounted` + `watch(events)` 在初始挂载时几乎同时触发 rebuildPlayer。
- rebuildPlayer 内 `await loadPack()` 期间 player 仍为 null,两次都跳过 `if (player)` 清理。
- 各自创建一个 Svelte Player → 两个 `.rr-player` 元素并存 → flex 各缩一半 → 视觉 bug。

**根因 2**(只在 bug 1 修复后暴露):1c 切片为旧 layout 加的 CSS hack `iframe { width: 100%; height: 100% }`。
- 现在父链 `.rr-player > .rr-player__frame > .replayer-wrapper` 都无显式高度。
- CSS `height: 100%` 无法 resolve,浏览器 fallback 到 iframe 默认 **150px**(HTML 规范)。
- 实测 iframe computed height = 150px(应 904px),视觉严重压扁。

**根因 3**(用户报告 5 条 sandbox warning → 实测 123 条):`initialized` flag 永远 false。
- watch callback 里 `if (player) initialized = true` 同步执行,但 `rebuildPlayer()` 是 async 返回 Promise。
- 同步检查时 player 还是 null,initialized 永远不被设 true。
- 每个 WS 事件都触发完整 rebuild → 新建 iframe → 浏览器报 sandbox warning。
- 30 秒访问实测:**123 条 sandbox warning + 24 条 "replayer destroyed"**。

**修复**:
- Bug 1:加 `rebuildToken` 计数器,await 后 `if (myToken !== rebuildToken) return` 放弃过时调用。把 `replaceChildren()` 移出 `if (player)` 守卫无条件清空。
- Bug 2:移除 `width/height: 100%` CSS。iframe 改用自身 attribute(1913×904)+ wrapper transform:scale 正确缩放。ReplayPlayer.vue + ReplayViewer.vue 两处都改。
- Bug 3:把 `initialized = true` 移到 rebuildPlayer 内部 player 创建成功后设置。watch callback 里删除无效的同步检查。

**验证**(Chrome DevTools MCP):
- `.rr-player` 数量 2 → 1 ✅
- iframe computed height 150px → 904px ✅
- iframe rect 607×84 → 1066×504 ✅
- sandbox warning 123 → 2(降 98%)✅
- "replayer destroyed" 24 → 0 ✅
- 截图 AI 视觉分析确认录屏清晰可见、大小合理、无过大空白 ✅
- 152 单测全绿无回归 ✅
