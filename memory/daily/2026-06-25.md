### 代码现状盘点（grill 前探索）

- 录制端 `visitor-sdk/src/collectors/rrweb.ts`：动态 import `rrweb` 的 `record()`，加了重试/visibility/周期 full snapshot 韧性。
- 回放端 `admin/src/components/ReplayPlayer.vue`：**重灾区**。~300 行约 200 行在和 Svelte 版 rrweb-player 内部搏斗——MutationObserver 补 iframe sandbox、live 模式 hack（`startLive` + 远未来 baseline）、响应式 sizing hack、双 mount 竞态 token、iframe display 强制覆盖。
- 节点定位 `nodeMap.ts`：靠扫 DOM 上 `data-rr-node-id` attribute（**假设存疑**——rrweb mirror 默认把 nodeID 存 JS Map 不写 DOM）。
- 协同点选 `CoBrowseOverlay.vue`（1f）：明确 MVP fallback，`nodeID=0` + 服务端按坐标点击；注释说"后续 monkey-patch rrweb-player"。
- 依赖锁 `2.0.0-alpha.20`：record / rrweb-snapshot（SDK）+ rrweb-player（admin）。
- 选择性截图是**自己的** collector（非 rrweb canvas）；全仓无 rrweb canvas 录制 / console plugin / packer / plugin 系统使用；传输走 `@msgpack/msgpack`。

### 11 条决策（详见总纲 spec）

1. 策略 = **硬分叉 A**（拷全 snapshot/record/replay，断联自维护）。
2. 包结构 = **单包 `@pinconsole/replay-core` 分 snapshot/record/replay/types**。
3. rrweb-player = **彻底删 Svelte，admin 原生 Vue 重写**直接持有 Replayer。
4. 抽取形态 = 拷上游 **TS 源码**（非 dist）。
5. 基线 = 钉死 **alpha.20**，先求行为等价不顺手升级。
6. 节奏 = **先复刻后雕刻**，两阶段隔离。
7. packer = 砍，压缩**下沉存储层 gzip**。
8. Shadow DOM / 同源 iframe = **保留**（访客站点不可预测）。
9. 新能力 = **①nodeID 跨端寻址 + ③删 nodeMap + ④Replayer 直控**，②元素原语紧随。
10. 上游测试 = **按保留面裁剪移植**（不全量）。
11. 分支 = **开 `feat/vendor-rrweb`**。

### 验证四防线

既有 e2e（1c/1d/1e/1f/1l）当验收门 + parity diff 夹具（喂同一事件流 diff 旧/新 Replayer 的 iframe outerHTML）锁阶段一等价 + 移植上游保留功能 jest 锁精简保真 + 新 nodeID 配正负向 e2e（陈旧 id → 优雅 no-op）。
