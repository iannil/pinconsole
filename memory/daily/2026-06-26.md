# 2026-06-26 每日笔记

## 修复：client 页面操作未在 admin 会话自动复现

### 根因
`ReplayPlayer.vue` 直接使用 `@pinconsole/replay-core` 的 `Replayer` 类（不经过 rrweb-player Svelte）。`Replayer` 构造后停留在 `paused` 状态，`baselineTime=0`，`timer` 未激活。`addEvent` 在 `paused` 状态下因 `isSync=false` 且 `timer.isActive()=false` 静默丢弃事件。

`replayer.on('finish', ...)` → `startLive(farFuture)` 方案依赖自动播放，但直用 Replayer 时不自动播放，`finish` 永不触发。

### 修复
`admin/src/components/ReplayPlayer.vue:88-95`：创建 Replayer 后立即调用 `replayer.startLive(Date.now() + 365d)`，将 `baselineTime` 设为远未来，启动 `timer`，使后续 `addEvent` 同步渲染。

### 额外修复（同批次）
- **useResponsivePlayerSize**：contain→cover 模式 + iframe 就绪重试机制（20×100ms）
- **visitors.ts store cap**：500→5000（防止 cap 裁切后 watch 不触发）
