## 核心业务形态

通过对竞品技术栈要求与业务经验要求的拆解，推断竞品是**ToB（面向企业/商户）的、带有强销售/客服转化属性的实时互动与监控系统**。

发帖人原话明确："**前端浏览器网页输入，后台实时显示，后台操作和浏览器网页互动**的一套 win+mac 的系统"。

具体形态：

- **实时监控 + 劫持式客服**：客户打开落地页（H5/Web），在输入框里**还没点击发送**的文字、鼠标停留的位置，后台销售/客服通过 Win/Mac 桌面端实时看到。客服可主动在后台操作，强制前端网页弹出优惠券、发起聊天、甚至控制网页跳转。这类工具在某些暴利行业的销售转化中非常吃香。
- **低代码互动营销平台**：商户通过后台的拖拽 / 可视化编辑器快速生成营销网页或表单。客户访问时通过 WebSocket 建立低延迟连接，进行实时互动。
- **高级协同浏览（Co-browsing）**：类似 LogRocket 或高级版客服系统，客服可实时接管用户的浏览器页面，协助用户完成复杂的表单填写或购买流程。

## 技术栈反推（竞品为什么需要这些？）

- **Golang + WebSocket（最高 500 同时在线）**：500 大概率指单房间/单商户节点的并发，而不是系统总并发。Go 处理高并发 WebSocket 有优势，适合做实时监控的信令流转。
- **Vue3 + 拖拽搭建/可视化编辑器**：系统不仅是监控，还提供"建站"功能。前端展示的网页是商户自己拖拽生成的（低代码平台经验）。
- **Tauri 桌面应用**：商户/客服端是安装在 Win/Mac 上的客户端，而非纯网页——通常是为了获得系统级通知、更高性能、防止浏览器标签被误关。
- **Redis 限流 + 网页防爬虫 + 防越权/防刷接口**：系统面临激烈竞争或恶意攻击。落地页被同行爬取，接口被恶意刷量。
- **私有化部署 + SaaS**：系统既可作为 SaaS 卖账号，也支持大客户买断部署到自己服务器。
