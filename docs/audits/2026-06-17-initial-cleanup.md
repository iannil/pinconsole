## 处理详情

### START.md 清理统计

- 清理前：约 122 行（10.7KB）
- 清理后预期：约 60-70 行
- 删除内容：招募帖原文 ~50 行
- 保留内容：业务上下文（约 10 行）+ 技术分析（约 50 行）

### CLAUDE.md 与 PLAN.md 的"重叠"

| 内容 | CLAUDE.md 中的角色 | PLAN.md 中的角色 |
|---|---|---|
| 架构决策清单 | 索引（一行一决策，禁止重新讨论） | 详情（决策 + 理由） |
| 技术栈 | 引用（参考 PLAN.md §4） | 完整说明 |
| 实施顺序 | 简述（始终从下一个子切片开始） | 1a-1j 完整表格 + 估时 |

**结论**：重叠是有意设计——CLAUDE.md 让 LLM 快速判定"这个决策不能动"，PLAN.md 让 LLM 理解决策细节。两者职责不同，无需去重。

### LICENSE 与 README 缺失

LICENSE 是法律文件，AGPL-3.0 是 PLAN.md 已决策项，必须在仓库根存在。本次审计补齐。

README.md 是 OSS 项目入口，包含项目简介、状态、文档导航。pre-code 阶段也应有，便于 LLM 与贡献者快速定位。
