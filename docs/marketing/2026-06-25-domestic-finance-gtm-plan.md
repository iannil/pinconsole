## TL;DR

把目标客户从"泛 ToB 决策者"**收窄到金融行业的远程陪访 / 共浏览 / 双录辅助的私有化场景**（监管强制、必须私有化、预算天然百万级）。只做 **50 万+、最好 100 万+** 的大单。因单人精力有限且零现成关系，**D 类 SI 渠道升为主战线**（出资质、销售、交付，你做上游产品 + 技术兜底 + 商用授权），B 类只保留大型 SaaS/大厂 OEM。冷启动靠**金融级信任包 + 一个压价灯塔案例**破冰。短期单点精准 outbound + 长期单条技术内容线复利，其余渠道全部不碰。现金牛主体 = **商用授权年费 + 年度维护订阅**。判活靠领先指标 + 三道闸门，18 个月判活窗口。

## 9 项根级决策

| # | 决策点 | 选择 | 理由 |
|---|---|---|---|
| 1 | 主转化目标 | 咨询交付（A）为主 + 商用授权（B）被动承接；北极星 = 持续收费现金牛 | OSS 本身免费，高单价只能来自服务；现金牛 = 必须有"会续的钱"，不是一锤子部署 |
| 2 | 目标客户 | 金融行业（保险/证券/银行 + 金融科技）私有化场景；D 类 SI 渠道为主战线，A 类金融经 D 撬，B 类只留大厂 OEM | 只有金融强监管刚需场景能稳定开百万预算；单人扛不下大单的招投标/资质/交付，必须借 SI |
| 3 | 对外楔子 | 主楔子 = 可白标可私有化的实时陪访 / co-browsing 中间件（buy-vs-build）；数据主权降为支撑弹药；不点名竞品 | B/SI 的真实痛点是"省自研 + 可私有化交付 + 商用授权清晰"；数据主权是他们转售给金融客户的弹药 |
| 4 | 获客引擎 | 哑铃：短期精准 outbound 拿现金 + 长期单条技术内容线复利；其余渠道不碰 | 单人最大死法是 5 个渠道一起半温；零关系打金融时，内容/信任线被迫前置为"近期就必须做" |
| 5 | 报价阶梯 | 商用授权年费 + 集成交付 + 强制年维护；**只做 50-100 万+** | 续费是现金牛命脉；精力有限只接大单 |
| 6 | 场景收窄 | 金融远程陪访 / 共浏览 / 双录辅助的私有化能力 | 监管强制 + 必须私有化 + 百万预算，正好是 co-browsing + 录像 + 数据主权三件套交集 |
| 7 | 冷启动破冰 | 0-3 月攒金融级信任包；灯塔案例走 (a) 金融边缘直客 + (b) 赌你的 SI 并行，(c) 内容养专家身份做底层；接受首个灯塔压价 | 金融不把百万项目交给零案例单人项目；灯塔是撬百万单的入场券 |
| 8 | 英语方案 | 待定 / 封存为并行选项，不砍，暂不执行 | 用户决定执行时机；与金融线人群/叙事零复用，并行会两条都半温 |
| 9 | 判活机制 | 三层领先指标 + 三道闸门，18 个月判活窗口；同一时间只压一条主战线 | 长周期零收入期里要能区分"还没到收获"和"路本来走不通" |

## 关键张力（已在访谈中显式解决）

1. **"只做 100 万+"与"主攻 B 类中型 SaaS 白标"冲突** → 重排：D 类 SI 渠道升为主战线，B 类只留大厂 OEM，场景收窄到金融。
2. **零关系 + 金融 + 单人 = 最难冷启动** → "长期内容/信任线"被迫前置为近期主线；先攒信任包再 outbound。
3. **冷启动信任悖论**（百万单需案例，案例需第一单）→ 接受压价灯塔案例（10-30 万），其真实价值是入场券而非这笔钱。
4. **资质门槛**（金融直采要供应商资质，单人没有）→ "幕后骑 SI 资质"既是销售通道也是资质通道，强化 D 渠道为主。

## 目标客户画像（收窄后）

### 主战线：D 类 SI / 集成商（有金融客户的）
- **特征**：手里有保险/证券/银行/金融科技的数字化项目；缺"实时陪访 / 共浏览 / 双录辅助"这块能力；有供应商资质和交付关系。
- **你对他的价值**：上游产品 + 技术兜底 + 可白标商用授权，让他不用自研就能在金融项目里补上这块能力。
- **续费点**：商用授权年费 + 年维护 + 项目分成。

### 次战线：B 类大型 SaaS / 大厂 OEM
- **特征**：自己服务金融政企客户，想给自家产品嵌入陪访能力，扛得起百万授权。
- **续费点**：白标商用授权年费（按部署实例 + 席位计）+ 年维护。

### 终端价值方：A 类金融机构（不直接打，经 D 撬）
- 保险（远程投保双录、客户陪访）、证券（远程开户双录）、银行（远程业务办理辅助）。
- 监管强制私有化、数据不出域——这是预算来源，但接触和交付都经 SI。

## 报价阶梯（只做大单版）

| 级 | 内容 | 价位锚点 | 续费属性 |
|---|---|---|---|
| 0. 免费自评 | 直接拉 AGPL 版自托管试跑（OSS = 免费试用品） | ¥0 | 转化诱饵 |
| 1. 商用授权年费 | 闭源嵌入自有产品 → AGPL 墙触发，必须买授权（按部署实例 + 席位） | **核心：单项目授权落在 50-100 万+ 的大盘里** | ✅ 年费，现金牛主体 |
| 2. 私有化集成交付 | 把陪访能力嵌进 SI/大厂产品 + 交付到金融客户内网 | 打包进百万级大单 | 一次性，带出级 3 |
| 3. 年度维护 + 升级订阅 | rrweb/安全/版本持续更新 + SLA 兜底 | 授权额的 15-20%/年 | ✅ 年费，第二条现金流 |
| D. 渠道授权 | SI 拿上游产品做交付，OEM 年费或项目分成 | 分成 15-30% 或 OEM 年费 | ✅ 可续 |

**铁律**：无续费项不成单（级 1 或级 3 必挂一个）。唯一例外是**首个灯塔案例**——允许压价（10-30 万）换可署名 case，但即便压价也尽量保留一个小额年维护以验证续费模型。

## 执行计划（短期 / 中期 / 长期）

### 短期（0-3 月）· 攒金融级信任包【主线，不急 outbound】

- [ ] **安全合规白皮书**：等保 2.0 就绪、数据不出域架构图、密评就绪、审计日志、《个人信息保护法》对齐、敏感操作授权链；把"**开源即可审计 + AGPL 不被云厂商拿走**"叙述为比闭源 SaaS 更强的金融信任点。
- [ ] **可现场私有化部署的 demo**：能在隔离/内网环境跑起来（不是公网试用沙盒——与金融"数据不出域"叙事一致）；预置一段金融陪访/双录辅助的演示流程。
- [ ] **官网定位重排**（改造 `marketing/`，详见下节）：主楔子转白标 buy-vs-build，新增 `/embed`（大厂 OEM）和 `/partners`（SI 渠道）入口，数据主权 Hero 降为 A 类/自用方次泳道。
- [ ] **启动长期内容线第一篇**（见长期节）。

### 中期（3-9 月）· 破冰找灯塔【outbound 开闸】

- [ ] **(a) 金融边缘直客 BD**：盯金融科技公司 / 保险经纪 / 中小券商（决策快、愿试新），手工建买信号名单（在招 rrweb/WebSocket/共浏览/双录相关岗位的公司、产品明显缺这块的金融科技 SaaS），1v1 定向触达，目标"署名案例 + 压价试点"。
- [ ] **(b) 物色赌你的 SI**：列 10-20 家有金融项目的 SI / 集成商，谈"白菜价授权换署名案例 + 后续优先合作"；明确 SI 是资质 + 销售通道。
- [ ] **目标**：进到 ≥3 个真实方案对话，启动 ≥1 个 POC/试点。

### 长期（持续）· 复利内容线 + 现金牛续费

- [ ] **单条技术内容线**：主题锁"私有化 co-browsing / 金融远程陪访 / 双录辅助怎么落地"系列，同步掘金 + 知乎 + 公众号，沉淀百度/必应 SEO；它同时被金融/SI 搜到 + 给你卖咨询当专业背书。**只养这一条，不开第二条。**
- [ ] **GitHub / Gitee 镜像打磨**：README 顶部金融私有化叙事 + demo 视频 + quickstart 可复现。
- [ ] **灯塔案例 → 撬百万单**：拿到署名 case 后，用它做 SI/大厂 OEM 的 BD 弹药，把单价从灯塔的压价水平拉回 50-100 万+。
- [ ] **现金牛运营**：每个成单强制挂年维护 + 授权年费，沉淀续费基本盘。

### 明确不碰
抖音 / 小红书 / 付费广告 / 线下展会 / 公网 demo 沙盒（与金融数据不出域叙事冲突）/ 并行铺多渠道 / 英语线（封存待定）。

## 官网（marketing/）改造点（相对 2026-06-22 设计）

- **主楔子重排**：Hero 从"你的访客，你的数据"（终端自用叙事）改造为"可白标、可私有化的实时陪访 / co-browsing 能力——省掉自研 rrweb 双向管道的 3-4 个月"。
- **数据主权降级为支撑弹药**：移到信任峰值段（架构 + 合规），并保留为"A 类/自用方"次泳道入口。
- **新增 `/embed`**（大厂 OEM）：给自家产品补齐可私有化白标陪访能力，CTA = 预约嵌入方案咨询。
- **新增 `/partners`**（SI 渠道）：上游产品 + 技术兜底 + 渠道分成/OEM 授权，CTA = 渠道合作洽谈。
- **新增金融信任区**：白皮书下载入口 + "开源即合规优势"叙事 + 等保/密评/个保法就绪清单。
- **保留**：不点名竞品（国内投诉文化风险）、留资表单中摩擦入口、Calm Crafted 视觉、`marketing/` 与 OSS `landing/` 严格区隔。

## 判活：领先指标 + 三道闸门（每 3 月体检）

| 阶段 | 时间 | 判活领先指标（非收入） | 不达标信号 |
|---|---|---|---|
| 信任包 | 0-3 月 | 白皮书 + 可私有化 demo 完成；技术内容有金融/SI 读者真实互动（评论/私信/收藏） | 内容零互动 = 叙事或人群错，改叙事不改方向 |
| 破冰 | 3-9 月 | ≥3 个真实方案对话；≥1 个 POC/试点启动 | 9 月 0 真实对话 = 信任包没说服力或触达方式错 |
| 灯塔 | 9-18 月 | 签下 1 个灯塔案例（可压价）；拿到可署名 case | 18 月 0 灯塔 = 场景/价格/资质模型需重审 |

**三道闸门**：
1. **3 月内容零互动** → 不否定金融方向，改叙事 / 换内容平台再试一轮。
2. **9 月 0 真实对话** → 触达方式失败，在"等人上门 ↔ 主动 BD SI 名单"之间切换打法。
3. **18 月 0 灯塔** → 真正该重审方向的点（场景是否够刚需、价格模型、是否必须先找主体挂资质）。

**产能护栏**：同一时间只压一条主战线（金融国内线）；英语线封存待定；不并行铺渠道；灯塔前不接与金融无关的杂活分散精力。

## Open Questions（执行前需补的决策）

1. **资质主体**：单人无金融供应商资质——是全程骑 SI 资质（永远幕后上游），还是早期就注册一个公司主体便于后续直签？影响 D 渠道依赖度。
2. **白皮书合规口径**：等保/密评/个保法的具体就绪声明需法务口径校对，避免过度承诺。
3. **内容线主笔精力**：技术深度内容是你亲自写还是有协作？这条线是长期复利的唯一引擎，断更即失效。
4. **耐心窗口**：本方案按 18 个月判活窗口设计；若你实际现金压力要求 12 个月内见钱，需把灯塔压价更狠 / 被迫先接小活养现金流（与"只做大单"冲突，需取舍）。
5. **英语线执行时机**：保留待定，但需明确"什么条件触发二次激活"（建议：国内线拿到首个灯塔 + 有余力后）。
